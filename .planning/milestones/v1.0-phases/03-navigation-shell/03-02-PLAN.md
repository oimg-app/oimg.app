## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user typing → Toolbar filter input → setFilter → filesAtom.filterQuery → $filteredFiles | User-controlled string flows into a substring filter; no eval, no innerHTML, no network. |
| user click → menu items / segments → store actions | All handlers are pre-bound to store actions; payloads are literal constants. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering / XSS | Toolbar filter input → file row rendering downstream | mitigate | `setFilter` writes the raw string into `filesAtom.filterQuery`. Downstream FileRow uses JSX text-node interpolation (already audited in Phase 2). This plan only adds the input — no new render surface. Acceptance guard: `grep -rc "dangerouslySet" src/components/shell/` returns 0. |
