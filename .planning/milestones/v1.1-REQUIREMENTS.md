## v1.1 Requirements — all satisfied

Reconnecting the real optimization pipeline behind the v1.0 UI. The v1.0 milestone built the full UI rendering from stub data; this milestone made it actually optimize images.

### Pipeline Infrastructure

- [x] **PIPE-01**: Optimization runs off the main thread via a Comlink-wrapped WorkerPool — the UI stays responsive (no jank) while files encode → **VALIDATED** (Phase 8; integration check verified `worker-pool.ts` → Comlink wrap of `codec.worker.ts`)
- [x] **PIPE-02**: Codecs are dynamically imported inside workers so the initial route stays < 200KB gzipped; AVIF loads only when the user selects it → **VALIDATED** (Phase 8; final bundle 194.88 KB gzipped, ~5 KB headroom; AVIF wasm split out as separate chunk)
- [x] **PIPE-03**: COOP/COEP response headers are configured so SharedArrayBuffer is available for multi-threaded codecs (OxiPNG, AVIF) → **VALIDATED** (Phase 8; dev guard in worker + Cloudflare `_headers` file)
- [x] **PIPE-04**: Backpressure is enforced — the pool bounds concurrent jobs and the BackpressureIndicator reflects real running state → **VALIDATED** (Phase 8 + Phase 11 SC-4 e2e proves peak ≤ Math.min(hwConc, 4) AND ≥ 2 on 20-file batch)

### Encoding

- [x] **ENC-01**: User can optimize a PNG via OxiPNG (decode through @jsquash/png, re-encode) and get real reduced-size output → **VALIDATED** (Phase 9)
- [x] **ENC-02**: User can encode/convert to WebP with quality and lossless controls → **VALIDATED** (Phase 9)
- [x] **ENC-03**: User can encode/convert to JPEG (MozJPEG) with quality and progressive controls → **VALIDATED** (Phase 9)
- [x] **ENC-04**: User can encode/convert to AVIF (lazy-loaded) with quality control → **VALIDATED** (Phase 9)
- [x] **ENC-05**: User can optimize an SVG via svgo with the inspector's plugin toggles actually applied → **VALIDATED** (Phase 9)
- [x] **ENC-06**: Inspector settings (quality, effort, lossless, resize, strip-metadata) drive the real encode output per codec → **VALIDATED** (Phase 9 + useLiveEncode debounce)

### Optimize Loop

- [x] **OPT-01**: User drops a single file → sees real optimized output with accurate before/after byte sizes in the Report → **VALIDATED** (Phase 10)
- [x] **OPT-02**: User clicks Optimize-all → the batch runs through the worker pool with live per-file progress → **VALIDATED** (Phase 11; streaming per-promise write-back + StatusBar X/Y counter)

### Export

- [x] **EXP-01**: User can download a single optimized file to disk → **VALIDATED** (Phase 11; showSaveFilePicker native + file-saver fallback; Inspector Download + FileRow Save as… + Toolbar Save individually)
- [x] **EXP-02**: User can export the entire optimized batch as a ZIP (jszip) → **VALIDATED** (Phase 11; timestamped `oimg-export-YYYY-MM-DD-HHMM.zip`, flat layout, collision-suffix, optimized-only)

### Snippets

- [x] **SNIP-01**: Output panel snippets (Base64 data-URI, URL-encoded, `<picture>`) reflect the real encoded bytes of the selected file, not stub placeholders → **VALIDATED code-side** (Phase 12; chokepoint exclusivity grep-confirmed; useEffect deps include `encodedBuffer`; 4 paste-into-real-browser dogfood checks captured for follow-up)
