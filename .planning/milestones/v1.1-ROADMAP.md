## Overview

Reconnects the real codec pipeline behind the v1.0 UI so a developer can drop assets, adjust settings, and walk away with real optimized files + ZIP + paste-ready snippets — all client-side. The v1.0 milestone shipped the UI scaffolding with stubs; v1.1 wires every panel to the actual jSquash + svgo encoding pipeline running off-main-thread in a bounded Comlink WorkerPool, then layers single-file download, batch ZIP export, and copy-paste-ready snippet output on top.

End-to-end user promise from PROJECT.md is now walked code-side:

> drop folder → settings once → real optimized result → ZIP + paste-ready snippets, nothing leaves the browser

## Milestone Summary

### Key Accomplishments

1. **Real codec pipeline running off-main-thread** — Comlink-wrapped WorkerPool with `Math.min(hwConc, 4)` cap; per-codec adapters (`@jsquash/oxipng`, `@jsquash/webp`, `@jsquash/jpeg` (MozJPEG), `@jsquash/avif` lazy-loaded, `svgo` v4 ESM) wired so inspector controls measurably shape encoded bytes.
2. **End-to-end optimize → download flow** — Drop a file → real encodedBuffer in Report; tweak quality → debounced useLiveEncode pushes new bytes; click Download → showSaveFilePicker (native) or file-saver (fallback) writes to disk with swapped extension.
3. **Batch optimize at scale** — `Optimize all` streams per-promise write-back; FileRow status flips live during the batch; StatusBar `X/Y optimized` counter advances mid-run; pool concurrency cap verified on 20-file batch (peak ≥ 2 AND ≤ Math.min(hwConc, 4)).
4. **Batch ZIP export via jszip** — Timestamped filename (`oimg-export-YYYY-MM-DD-HHMM.zip`), flat layout, collision-suffix `(1)/(2)/…`, optimized-only, error-skip with toast, T-11-01 zip-slip mitigation via `sanitizeBaseName`.
5. **Paste-ready snippets reflect real bytes** — Three Output panel sections (Base64 data-URI, URL-encoded, `<picture>`) + three Toolbar bulk actions (Copy `<picture>` HTML, Copy as data URIs, Manifest JSON) + two FileRow row items (Copy `<picture>`, Copy data-URI), all routing through one clipboard chokepoint with `navigator.clipboard.writeText` → textarea+execCommand fallback.
6. **Security hardening across the milestone** — T-11-01 zip-slip (`sanitizeBaseName` applied at every JSZip.file name AND showSaveFilePicker suggestedName); T-12-01 SVG XSS (URL-encoded path always wins for `target === 'svg'`); T-12-02 HTML-attr injection (`escapeAttr` on alt/src/srcset); T-12-03/T-12-04 clipboard fallback safety (secure-context gate + try/finally textarea cleanup); T-WF-01..04 directory watching (secure-context, existing-pool reuse, observer.disconnect, permission-revoked clear).

### Key Decisions

- **WorkerPool API extended with optional `onDispatch?` hook** (Phase 11) — non-breaking for Phase 9 codec adapters; enables Phase 11's streaming write-back to observe dispatch-time vs await-completion-time. Validated via Phase 12 + quick task — no regressions.
- **Chokepoint pattern** — Phase 11 established `save-blob.ts` as the only `showSaveFilePicker` caller; Phase 12 extended the pattern with `clipboard.ts` as the only `navigator.clipboard.writeText` caller; quick task added `dir-picker.ts` as the only `showDirectoryPicker` caller. Single funnel for feature-detection + AbortError handling + toast UX.
- **Disable-then-explain over interstitial dialogs** — Phase 11 D-13 `$hasDone` computed atom drives `disabled` + `aria-disabled` + `title` on Toolbar Export controls; Phase 12 reused verbatim for Toolbar bulk snippet items AND FileRow row items. WCAG-AA inherited from Radix primitives.
- **No new state atoms when derivation suffices** — StatusBar X/Y counter, OutputPanel snippets, `$hasDone` all derive from existing `filesAtom` + `runtimeAtom`. No `batchProgressAtom`, no `snippetsAtom`.
- **Yoksel-style minimal URL-encoding for SVG data URIs** — selective unescape of `( ) ! ~ *` for CSS-paste compatibility, re-escape of `'` and `"` for back-compat. Raster falls back to chunked-32KB base64 (avoids V8 call-stack blowup at ~125KB).
- **Quick tasks live alongside phases** — `260603-s2x` Watch folder shipped a full 8-task plan + threat model + tests + STATE.md table update in one session, demonstrating the quick-task lane works for non-trivial scope.
