# Requirements: oimg.app — v1.2 Polish, Diagnostics, PWA + Quality Metrics

**Defined:** 2026-06-10
**Core Value:** A developer drops assets, adjusts settings once, and walks away with optimized files + ZIP + paste-ready snippets — with real perceptual quality scores, installable as a desktop/mobile PWA, fully offline-capable on second visit. Nothing leaves the browser.

## v1.2 Requirements

Layers polish + diagnostic transparency + perceptual quality + installable/offline distribution on top of the v1.1 real codec pipeline.

### Ingest

- [ ] **ING-01**: Toolbar "From URL or paste" wires the previously-empty `addFromUrl` stub. On click, reads `navigator.clipboard.read()` (or paste-event `clipboardData.items` as fallback). Detects: (a) image bytes directly in clipboard → ingest via `useIngest.ingest`, toast "Pasted from clipboard: {name}"; (b) image URL as plain text → fetch directly, ingest, toast "Imported from URL: {host}"; (c) anything else → toast "Clipboard has no image or image URL". CORS-blocked URL → clear toast "URL blocked by CORS — download and drop the file, or paste it directly."
- [ ] **ING-02**: Document-level paste event handler on the app root: pasting an image directly anywhere in the app (Cmd/Ctrl+V) ingests it through the same dispatcher as ING-01. Same toasts.

### Queue Hygiene

- [x] **CLR-01**: `clearFiles()` action added to `src/stores/files.ts`. Toolbar overflow menu gains "Clear all" item. FilesPane header gains a small × icon. Both call the same action. Both disable-then-explain (Phase 11 D-13 pattern) when queue is empty.

### Quality Metrics

- [ ] **MTR-01**: `ssim.js@3.5.0` integrated as the SSIM comparator. Runs in the codec worker (or a sibling metrics worker) post-encode. Auto-triggers for the **selected file** only when its `status === 'done'`. Result cached on `FileEntry.metrics.ssim`. Refreshes on re-encode via Phase 9 `useLiveEncode` trigger. Lazy-loaded — does NOT enter the initial bundle.
- [ ] **MTR-02**: Hand-built wasm Butteraugli (Emscripten compile of Google libjxl's butteraugli comparator) integrated alongside SSIM. Same trigger rules (selected file, post-`done`, refresh on re-encode). Result cached on `FileEntry.metrics.butteraugli`. Lazy-loaded — separate dynamic-import chunk.
- [ ] **MTR-03**: Report panel displays both metrics with banded coloring: green (good), yellow (acceptable), red (poor). Thresholds: SSIM green ≥ 0.95, yellow ≥ 0.85, red < 0.85; Butteraugli green < 1.5, yellow < 3.0, red ≥ 3.0 (lower is better per Google's conventions). Both thresholds are documented constants editable in code.

### Diagnostics

- [x] **DIA-01**: `versionsAtom` (or `diagAtom` if a new atom is preferred) added to `src/stores/runtime.ts`. Populated at build time via Vite `define`: `svgo` (from svgo's package.json), `@jsquash/*` versions (webp, jpeg, avif, oxipng, png, resize), `ssim.js`, butteraugli build hash.
- [x] **DIA-02**: Runtime capability detection on app boot — SIMD support (WebAssembly.validate test), shared memory + threads support, `crossOriginIsolated`, `navigator.hardwareConcurrency`. Cached in the same atom under `caps`.
- [x] **DIA-03**: StatusBar hardcoded version badge text ("svgo 4.0.1 · webp 1.5" or similar) replaced with live `versionsAtom` reads. Footer "100% local · Offline-ready" derives the "Offline-ready" segment from `navigator.serviceWorker.controller != null && SW.precacheComplete === true` — only shows when the SW is registered AND the app shell precache is complete.
- [x] **DIA-04**: Settings popover (Toolbar) gains a "Diagnostics" tab rendering the full versions + caps + SW status surface. Read-only, copy-to-clipboard (via Phase 12 `copyToClipboard` chokepoint) for bug-report convenience.

### Installable PWA

- [x] **PWA-01**: `manifest.webmanifest` generated by `vite-plugin-pwa@1.3.0` (injectManifest mode) with: name "oimg.app — Image Optimizer", short_name "oimg", icons (`/oimg-logo.svg`, `/oimg-logo-1024.png`, `/oimg-logo-maskable-512.png` — last to be added), theme_color `#5eb87a` (matches existing index.html), background_color matching dark theme, display "standalone", start_url "/".
- [x] **PWA-02**: Hand-rolled `src/sw.ts` using Workbox primitives (`@workbox/precaching`, `@workbox/routing`, `@workbox/strategies`). Precaches: app shell (index.html, JS, CSS, SVG logo). Runtime-caches: codec wasm files via `CacheFirst` strategy keyed on URL. AVIF's 3.4 MB wasm does NOT load on first visit — only on first AVIF use, then served from cache thereafter.
- [x] **PWA-03**: `beforeinstallprompt` handler in `src/main.tsx` or a `useInstallPrompt` hook captures the deferred prompt. When fired, StatusBar surfaces a small "Install" button near the offline pip. Click → call the deferred prompt. On install success → button disappears. Cross-browser: works in Chrome/Edge; Firefox does not fire the event (button never appears — no fallback needed); Safari iOS handled via meta-tag for "Add to Home Screen".
- [x] **PWA-04**: Cloudflare Pages `_headers` updated with: `/sw.js: Cache-Control: no-cache`, `/manifest.webmanifest: Cache-Control: public, max-age=86400`, plus the existing COOP/COEP headers preserved (SharedArrayBuffer for MT codecs).
- [x] **PWA-05**: Service worker handles its own version-bump skipWaiting + clientsClaim so users get updates without manual refresh. A "New version available — reload?" toast surfaces when a new SW takes over.
