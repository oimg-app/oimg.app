# v1.2 Research — Diagnostics Panel + Settings Popover

**Goal:** Replace the hardcoded `svgoVersion: '4.0.1'` / `codecVersion: '0.6.0'` / `wasmInfo: 'WASM ready · 312 KB'` strings in `runtimeAtom` with **real values read from the build**, and extend the existing Settings popover into a full diagnostics panel surfacing WASM features, hardware concurrency, COOP/COEP status, and worker count.

**Confidence:** HIGH on Vite version-injection mechanism, HIGH on feature detection.

---

## TL;DR — Recommended Path

1. **Read package.json versions at build time** via Vite's `define` option (zero runtime cost, no async import).
2. **Detect WASM features at runtime** (SIMD, threads, OffscreenCanvas, crossOriginIsolated) once on app boot.
3. **Replace the existing Settings popover** in `Toolbar.tsx` with a richer `SettingsPanel` component that uses the existing `Popover` Radix primitive (no new dependency). Lazy-load the panel via `React.lazy` so it stays out of the initial chunk.
4. **Add `versionsAtom` + `capabilitiesAtom`** to `src/stores/runtime.ts` (same file as `runtimeAtom`).

---

## 1. Version Injection — Vite `define`

Vite's `define` evaluates expressions at build time and inlines literal strings. Zero runtime, no JSON parsing, tree-shakeable.

`vite.config.ts`:

```ts
import pkg from './package.json' with { type: 'json' }

export default defineConfig({
  define: {
    __APP_VERSIONS__: JSON.stringify({
      app: pkg.version,
      svgo: pkg.dependencies['svgo'],
      jsquash: {
        webp:    pkg.dependencies['@jsquash/webp'],
        jpeg:    pkg.dependencies['@jsquash/jpeg'],
        avif:    pkg.dependencies['@jsquash/avif'],
        oxipng:  pkg.dependencies['@jsquash/oxipng'],
        png:     pkg.dependencies['@jsquash/png'],
        resize:  pkg.dependencies['@jsquash/resize'],
      },
      squooshKit: {
        imagequant: pkg.dependencies['@squoosh-kit/imagequant'],
        png:        pkg.dependencies['@squoosh-kit/png'],
        mozjpeg:    pkg.dependencies['@squoosh-kit/mozjpeg'],
        webp:       pkg.dependencies['@squoosh-kit/webp'],
        avif:       pkg.dependencies['@squoosh-kit/avif'],
      },
      react:   pkg.dependencies['react'],
      vite:    pkg.devDependencies['vite'],
    }),
    __BUILD_DATE__: JSON.stringify(new Date().toISOString()),
    __BUILD_COMMIT__: JSON.stringify(process.env.COMMIT_SHA ?? 'dev'),
  },
})
```

Add a `src/vite-env.d.ts` declaration:

```ts
declare const __APP_VERSIONS__: {
  app: string
  svgo: string
  jsquash: { webp: string; jpeg: string; avif: string; oxipng: string; png: string; resize: string }
  squooshKit: { imagequant: string; png: string; mozjpeg: string; webp: string; avif: string }
  react: string
  vite: string
}
declare const __BUILD_DATE__: string
declare const __BUILD_COMMIT__: string
```

The version strings are inlined as constants at build — bundle impact is just the strings themselves (~300 bytes gzipped for the full version dict).

**Caveat:** `pkg.dependencies['svgo']` includes the caret (`^4.0.1`). Strip with `.replace(/^[\^~]/, '')` if you want the pure version. Or accept the spec — surfacing `^4.0.1` is more honest than `4.0.1` (we haven't actually probed the lockfile).

**Alternative considered + rejected:** `import('svgo/package.json')` at runtime. Works in dev, breaks in prod (Vite doesn't bundle package.json files outside the public path). The `define` approach is more reliable.

## 2. Capability Detection (`capabilitiesAtom`)

Detect once at app boot. All synchronous except WASM-SIMD/threads (which need a tiny WASM probe).

```ts
// src/lib/capabilities.ts
export interface Capabilities {
  crossOriginIsolated: boolean
  offscreenCanvas: boolean
  showSaveFilePicker: boolean
  showDirectoryPicker: boolean
  fileSystemObserver: boolean
  webWorkerType: boolean       // module workers
  serviceWorker: boolean
  hardwareConcurrency: number
  wasm: { simd: boolean; threads: boolean }
  installPromptAvailable: boolean    // detected lazily by useInstallPrompt
}

// SIMD probe — 47-byte WASM module that uses v128.const
const SIMD_PROBE = new Uint8Array([
  0,97,115,109,1,0,0,0,1,5,1,96,0,1,123,3,2,1,0,
  10,10,1,8,0,65,0,253,15,253,98,11,
])

// Threads probe — requires SharedArrayBuffer (already gated by crossOriginIsolated)
async function detectWasmFeatures() {
  let simd = false
  try { simd = WebAssembly.validate(SIMD_PROBE) } catch { /* noop */ }
  const threads = typeof SharedArrayBuffer !== 'undefined' && self.crossOriginIsolated === true
  return { simd, threads }
}

export async function detectCapabilities(): Promise<Capabilities> {
  const wasm = await detectWasmFeatures()
  return {
    crossOriginIsolated: self.crossOriginIsolated === true,
    offscreenCanvas: 'OffscreenCanvas' in window,
    showSaveFilePicker: 'showSaveFilePicker' in window,
    showDirectoryPicker: 'showDirectoryPicker' in window,
    fileSystemObserver: 'FileSystemObserver' in window,
    webWorkerType: true,
    serviceWorker: 'serviceWorker' in navigator,
    hardwareConcurrency: navigator.hardwareConcurrency ?? 4,
    wasm,
    installPromptAvailable: false,  // flips true when useInstallPrompt captures BIP
  }
}
```

## 3. Store Extensions (`src/stores/runtime.ts`)

```ts
import { atom } from 'nanostores'
import type { Capabilities } from '@/lib/capabilities'

// Versions are build-time constants — pure atom for consistency with capabilities
export const versionsAtom = atom(__APP_VERSIONS__)
export const buildInfoAtom = atom({ date: __BUILD_DATE__, commit: __BUILD_COMMIT__ })

export const capabilitiesAtom = atom<Capabilities | null>(null)

export async function initCapabilities(): Promise<void> {
  const { detectCapabilities } = await import('@/lib/capabilities')
  capabilitiesAtom.set(await detectCapabilities())
}
```

## 5. Bundle Budget Impact

| Addition | Initial chunk gzip est. | Notes |
|---|---|---|
| `versionsAtom` + version dict | ~0.3 KB | Just a JSON literal |
| `capabilitiesAtom` + `initCapabilities` | ~0.1 KB | Atom is tiny; detection lib is dynamic import |
| `capabilities.ts` (detection) | 0 KB (dynamic) | Lazy-loaded via `initCapabilities` |
| `SettingsPanel.tsx` | 0 KB (lazy) | Behind `React.lazy` boundary |
| `useInstallPrompt` | ~0.4 KB | Tiny |
| **Total** | **~0.8 KB** | Safe within 5 KB headroom |

**Combined with quality-metrics + PWA + URL-paste budget impact (~3 KB):** total v1.2 cost ≈ 4 KB initial chunk. Under the 5 KB headroom; squeaks in.

If we go over budget during integration, the cheapest evict is the `wasmInfo` derivation in `StatusBar.tsx` — drop the capabilities readout from the bar and keep it only in the SettingsPanel.

## 6. Migration Plan (clean retirement)

1. Add `versionsAtom` + `capabilitiesAtom` + `buildInfoAtom` to `runtime.ts`.
2. Remove `svgoVersion`, `codecVersion`, `wasmInfo` from `RuntimeState` interface.
3. Update `StatusBar.tsx` consumers to read from `versionsAtom` + `capabilitiesAtom`.
4. Call `initCapabilities()` from `App.tsx` once on mount (inside the same `requestIdleCallback` as SW registration to avoid blocking first paint).
5. Build the `SettingsPanel` component; replace inline Settings popover content.
6. Smoke test: bundle stays under 200 KB gzipped.
