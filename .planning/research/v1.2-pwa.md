# v1.2 Research — PWA (Manifest + Service Worker + Install Prompt)

**Goal:** Make oimg.app installable and offline-capable on second visit, **without** ballooning the first-visit bandwidth. Codec WASMs (especially AVIF at 3.4 MB) must precache **opportunistically**, not on first load.

**Confidence:** HIGH.

---

## TL;DR — Recommended Strategy

1. **Plugin:** `vite-plugin-pwa@^1.0` (latest 1.3.0, 2025-07-09) using **`injectManifest` strategy** with a hand-rolled service worker (`src/sw.ts`). Workbox under the hood.
2. **Precache (first visit):** Only the app shell — `index.html`, the initial JS chunk, fonts, design tokens CSS, and the small logo. ~250 KB gzipped total. **Codec WASMs are NOT precached.**
3. **Runtime cache (opportunistic):** Use Workbox `CacheFirst` for `*.wasm` and codec chunk JS — the *first time* a codec is used, the SW caches it. Second visit → offline-ready.
4. **Manifest:** Static `public/manifest.webmanifest` referencing existing `oimg-logo.svg` + `oimg-logo-1024.png`. Maskable icon variant required.
5. **Install prompt:** Capture `beforeinstallprompt` in a `useInstallPrompt` hook; show an unobtrusive "Install" button in the Settings popover. Cross-browser limitations documented below.

Why **`injectManifest`** over `generateSW`: we already have one worker (`codec.worker.ts`) and Vite worker chunking. We need precise control over the runtime caching rules and the install-prompt path, plus the freedom to add a "skip waiting" prompt later. `generateSW` is too magic for a codec-heavy app.

## 2. vite-plugin-pwa Config

`vite.config.ts` addition:

```ts
import { VitePWA } from 'vite-plugin-pwa'

export default defineConfig({
  plugins: [
    // ...react, tailwind...
    VitePWA({
      strategies: 'injectManifest',
      srcDir: 'src',
      filename: 'sw.ts',
      registerType: 'autoUpdate',
      injectRegister: false,                       // we register manually so we can show a toast on update
      manifest: false,                             // we ship public/manifest.webmanifest by hand
      devOptions: { enabled: false },              // do NOT enable in dev — interferes with HMR
      injectManifest: {
        // Precache app shell only — exclude codec wasms (we runtime-cache them)
        globPatterns: ['**/*.{js,css,html,svg,woff2}'],
        globIgnores: ['**/node_modules/**', '**/*.wasm', '**/codec.worker-*.js'],
        maximumFileSizeToCacheInBytes: 1024 * 1024, // 1 MB ceiling for precache entries
      },
    }),
  ],
})
```

**Dev mode note:** `devOptions.enabled: false` is deliberate. The plugin's dev mode registers a SW in dev which interferes with Vite HMR. Test PWA behavior via `npm run build && npm run preview`.

## 3. Service Worker (`src/sw.ts`)

Hand-rolled with Workbox:

```ts
/// <reference lib="webworker" />
import { precacheAndRoute, cleanupOutdatedCaches } from 'workbox-precaching'
import { registerRoute } from 'workbox-routing'
import { CacheFirst, StaleWhileRevalidate } from 'workbox-strategies'
import { ExpirationPlugin } from 'workbox-expiration'
import { CacheableResponsePlugin } from 'workbox-cacheable-response'

declare const self: ServiceWorkerGlobalScope

// 1) App shell — injected at build time by vite-plugin-pwa
precacheAndRoute(self.__WB_MANIFEST)
cleanupOutdatedCaches()

// 2) Opportunistic codec WASM cache — CacheFirst (immutable hashed filenames)
registerRoute(
  ({ url, request }) => url.pathname.endsWith('.wasm') || request.destination === 'script' && url.pathname.includes('codec.worker'),
  new CacheFirst({
    cacheName: 'oimg-codecs-v1',
    plugins: [
      new CacheableResponsePlugin({ statuses: [0, 200] }),
      new ExpirationPlugin({ maxEntries: 30, maxAgeSeconds: 60 * 60 * 24 * 90 }),  // 90 days
    ],
  }),
)

// 3) Lazy codec chunks — runtime-cache after first hit
registerRoute(
  ({ url }) => /\/(jsquash|squoosh-kit)\//.test(url.pathname),
  new CacheFirst({ cacheName: 'oimg-codec-js-v1' }),
)

// 4) Fonts — StaleWhileRevalidate (small, frequently updated only on app version bump)
registerRoute(
  ({ request }) => request.destination === 'font',
  new StaleWhileRevalidate({ cacheName: 'oimg-fonts-v1' }),
)

self.addEventListener('install', () => self.skipWaiting())
self.addEventListener('activate', (e) => e.waitUntil(self.clients.claim()))
```

**Critical:** The codec WASMs are hashed by Vite at build (e.g. `avif_enc-abc123.wasm`). CacheFirst on hashed filenames is safe because a new build emits a new hash → new cache key → old cache entry is orphaned (ExpirationPlugin cleans it).

## 4. SW Registration (`src/lib/register-sw.ts`)

```ts
// Code-split — imported dynamically from App.tsx after first paint
import { registerSW } from 'virtual:pwa-register'
import { toast } from 'sonner'

export function bootstrapSW(): void {
  if (!('serviceWorker' in navigator)) return
  const updateSW = registerSW({
    onNeedRefresh() {
      toast('A new version is ready', {
        action: { label: 'Reload', onClick: () => updateSW(true) },
        duration: Infinity,
      })
    },
    onOfflineReady() {
      toast.success('Ready to work offline')
    },
  })
}
```

App.tsx wires this **after** initial mount (idle callback) so SW registration doesn't compete with first paint:

```ts
useEffect(() => {
  const id = requestIdleCallback?.(() => import('@/lib/register-sw').then(m => m.bootstrapSW()))
    ?? setTimeout(() => import('@/lib/register-sw').then(m => m.bootstrapSW()), 2000)
  return () => (typeof id === 'number' ? clearTimeout(id) : cancelIdleCallback(id))
}, [])
```

This keeps `register-sw` and `virtual:pwa-register` out of the initial chunk.

## 5. Install Prompt — `useInstallPrompt`

```ts
// src/hooks/useInstallPrompt.ts
import { useEffect } from 'react'
import { atom } from 'nanostores'
import { useStore } from '@nanostores/react'

interface BIPEvent extends Event { prompt(): Promise<void>; userChoice: Promise<{ outcome: 'accepted' | 'dismissed' }> }

export const installPromptAtom = atom<BIPEvent | null>(null)
export const isInstalledAtom = atom<boolean>(
  typeof window !== 'undefined' &&
  window.matchMedia?.('(display-mode: standalone)').matches
)

export function useInstallPrompt() {
  const event = useStore(installPromptAtom)
  const installed = useStore(isInstalledAtom)

  useEffect(() => {
    const onBIP = (e: Event) => { e.preventDefault(); installPromptAtom.set(e as BIPEvent) }
    const onInstalled = () => { isInstalledAtom.set(true); installPromptAtom.set(null) }
    window.addEventListener('beforeinstallprompt', onBIP)
    window.addEventListener('appinstalled', onInstalled)
    return () => {
      window.removeEventListener('beforeinstallprompt', onBIP)
      window.removeEventListener('appinstalled', onInstalled)
    }
  }, [])

  async function promptInstall(): Promise<boolean> {
    if (!event) return false
    await event.prompt()
    const { outcome } = await event.userChoice
    installPromptAtom.set(null)
    return outcome === 'accepted'
  }

  return { canInstall: !!event && !installed, installed, promptInstall }
}
```

Wire into Toolbar Settings popover:
```tsx
const { canInstall, promptInstall } = useInstallPrompt()
{canInstall && <button onClick={() => void promptInstall()}>Install app</button>}
```

### Cross-browser Install Matrix

| Browser | `beforeinstallprompt` | Native install UI | Notes |
|---|---|---|---|
| Chrome (desktop + Android) | YES | menu badge | Full support |
| Edge (desktop) | YES | menu badge | Full support |
| Firefox (desktop) | NO | — | Has no PWA install; SW + manifest still work for offline |
| Firefox (Android) | NO | — | Can "Add to Home Screen" manually |
| Safari (macOS) | NO | YES (Add to Dock, 17+) | No prompt API; user must use Share → Add to Dock |
| Safari (iOS) | NO | YES (Add to Home Screen) | User-driven flow only |

**Implication:** The "Install app" button only renders on Chromium browsers. Firefox/Safari users still get offline-on-second-visit because the SW + manifest work regardless of the prompt API.

## 6. Bundle Budget Impact (PIPE-02 carry-forward)

| Addition | Initial chunk impact | Why |
|---|---|---|
| `virtual:pwa-register` | 0 KB | Dynamically imported via `requestIdleCallback` |
| `register-sw.ts` | 0 KB | Same — lazy chunk |
| `useInstallPrompt` | ~0.4 KB gzip | Tiny; safe to ship in initial chunk for the Settings button to react instantly |
| SW itself (`sw.ts` + workbox-*) | 0 KB initial / ~15 KB gzip as SW | Loads as a separate file from `/sw.js`, not part of any app chunk |

**Headroom preserved.** Initial chunk stays under 200 KB gzipped.

## 7. Cloudflare Pages Caveats

- Cloudflare Pages serves `/sw.js` from root by default — vite-plugin-pwa emits it there.
- `_headers` already exists; verify COOP/COEP headers also apply to `/sw.js` (they should — they're set at root scope).
- The SW must NOT be cached by Cloudflare CDN beyond a few seconds — add `Cache-Control: max-age=0, must-revalidate` for `/sw.js` in `_headers`:
  ```
  /sw.js
    Cache-Control: max-age=0, must-revalidate
  /manifest.webmanifest
    Cache-Control: max-age=3600
  ```
