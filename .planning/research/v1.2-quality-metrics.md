# v1.2 Research — Quality Metrics (Butteraugli + SSIM)

**Goal:** Surface a per-file perceptual-quality readout in the Compare/Report view so users see *how much* visual quality they traded for the bytes saved. The readout must be lazy-loaded (initial route stays under 200 KB gzipped), run in the existing codec worker, and never block the encode pipeline.

**Confidence:** MEDIUM-HIGH on path, HIGH on bundle implications.

---

## TL;DR — Recommended Path

**Use SSIM from `ssim.js` v3.5.0** as the canonical metric for v1.2. **Defer Butteraugli to v1.3+**.

Rationale:
1. `@jsquash/butteraugli` does not exist on npm (404 — verified `npm view`).
2. No first-party WASM bundle of `google/butteraugli` ships standalone. The only browser-runnable copies are inside Squoosh's own bundle (Apache 2.0) and inside libjxl's encoder (large, encode-coupled). Extracting just the comparator is real engineering work, not a library install.
3. `ssim.js` is pure JS, ~30 KB minified (likely <10 KB gzip), tree-shakeable, MIT, last published 2025-07-08 (3.5.0). It works on `ImageData` directly — same shape our worker already produces.
4. SSIM is the cheaper, well-understood metric. It gets us to "we ship a quality readout" inside the bundle budget; Butteraugli is a v1.3 follow-up if users ask for it.

## 2. Runtime API Contract

Add to `src/workers/codec.worker.ts` alongside `optimize()`:

```ts
export interface QualityResult {
  ssim: number       // 0..1, higher = closer to original
  mssim?: number     // mean SSIM if multi-scale enabled
}

async function computeQuality(
  originalBytes: ArrayBuffer,
  encodedBytes: ArrayBuffer,
  format: 'png' | 'jpeg' | 'webp' | 'avif',  // SVG opt-out — quality metrics are raster-only
): Promise<QualityResult> {
  // Lazy import — keeps initial chunk clean
  const [{ default: ssim }, original, encoded] = await Promise.all([
    import('ssim.js'),
    decodeSource(originalBytes, format),
    decodeSource(encodedBytes, format),
  ])
  if (!original || !encoded) throw new Error('Decode failed for SSIM input')
  // ssim.js wants {data, width, height}; ImageData satisfies that shape.
  const { mssim } = ssim(original, encoded, { downsample: 'fast' })
  return { ssim: mssim }
}

Comlink.expose({ optimize, computeQuality })
```

Call site (new `useQualityMetrics` hook, batched after `setFileResult`):

```ts
// src/hooks/useQualityMetrics.ts
export async function computeAndStoreQuality(entry: FileEntry): Promise<void> {
  if (entry.type === 'svg') return
  if (!entry.rawBuffer || !entry.encodedBuffer) return
  const pool = getPool()
  const { ssim } = await pool.compute(entry.rawBuffer.slice(0), entry.encodedBuffer.slice(0), entry.type)
  setFileQuality(entry.id, { ssim })
}
```

Store extension (`stores/files.ts`):

```ts
interface FileEntry {
  // ...existing...
  quality?: { ssim: number; butteraugli?: number }
}
export function setFileQuality(id: string, q: { ssim: number }): void {
  updateEntry(id, () => ({ quality: q }))
}
```

WorkerPool extension (`src/lib/worker-pool.ts`): add a `compute(originalBuf, encodedBuf, format)` method that mirrors `run()` but calls `worker.computeQuality(...)`. Two Comlink.transfer payloads.

## 3. Bundle Impact

Initial route is at **194.88 KB gzipped** (5 KB headroom). All ssim.js code stays out of the initial chunk because:

- Worker file is its own chunk (`codec.worker-*.js`, currently 11.15 KB).
- Dynamic `import('ssim.js')` inside the worker case will be split into a separate worker-side chunk.
- `useQualityMetrics` hook lives in the inspector/report panel — already lazy-loaded if we put it behind the same lazy boundary as `CompareStage` / `ReportPanel` route.

**Risk:** if `useQualityMetrics` is imported eagerly at app boot (e.g. from `Toolbar` or `useOptimize`), it pulls ssim.js into the initial chunk. **MUST gate behind a dynamic import or behind the Report/Compare panel mount.**

Acceptance test: `npm run test:bundle` must still report initial chunk ≤ 200 KB gzipped after wiring.

## 4. SSIM Behavior Notes

- `ssim.js` accepts `ImageData`-shaped objects (`{ data: Uint8ClampedArray, width, height }`).
- Default `options.ssim = 'fast'` is the right tradeoff for live encoding (uses downsampling).
- Returns `{ mssim, performance, ssim_map }`. Use `mssim` (mean SSIM) as the single user-facing number. Drop `ssim_map` (heatmap is a v1.3 feature if we want a per-region visualization).
- Dimensions of original and encoded must match. **If the user resized**, we should compare encoded against the resized original, not the raw original. Easiest: compute the resized `ImageData` in the encode path, cache it under `entry.resizedBuffer` (or recompute via `maybeResize`), and pass that as the reference.

2. Alternative: vendor Squoosh's `codecs/butteraugli/` build under Apache 2.0 (license-compatible with our MIT — must add NOTICE).
3. Wrap with the same `computeQuality` interface — return `{ ssim, butteraugli }` and the UI surfaces the available metric(s).
