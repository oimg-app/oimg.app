# Quick Task 260603-s2x: Watch folder

**Goal:** Wire the Toolbar "Watch folder" menu item (currently calling the empty `addWatchFolder()` stub) to a real File System Access API flow: user picks a directory via `showDirectoryPicker`, all images in it are ingested via the existing `useIngest` pipeline, and on supporting browsers (Chrome 132+) a `FileSystemObserver` auto-ingests new image files added to the folder while the app is open.

**Why now:** Phase 10 explicitly deferred this (Phase 10 CONTEXT.md `<deferred>`, REQUIREMENTS.md milestone v1.0 NAV-02). v1.1 shipped without it. Landing as a quick task because the scope is small (one hook + dispatcher + stub replacement) and the ingest pipeline already exists.

## Scope

### In scope
- `src/lib/dir-picker.ts` — `pickDirectory()` dispatcher: feature-detect `'showDirectoryPicker' in window && window.isSecureContext`; on missing API, toast "Watch folder requires Chrome/Edge — your browser does not support directory access" and return null. AbortError on user-cancel → silent (mirrors Plan 11-04 save-blob.ts pattern).
- `src/hooks/useWatchFolder.ts` — hook exposing `startWatching(): Promise<void>` and `stopWatching(): void`.
  - `startWatching()`: stops any active observer; calls `pickDirectory()`; on success, iterates the directory handle's entries (one level deep; no recursion in this iteration); collects `File` objects whose extension matches `isAccepted` (export from useIngest); calls `useIngest.ingest(files)`; emits toast "Watching {dirName} ({N} files imported)".
  - If `'FileSystemObserver' in window`: instantiate observer, observe the handle, on each `appeared` or `modified` mutation type for a `FileSystemFileHandle` whose name passes `isAccepted`, call `handle.getFile()` then `useIngest.ingest([file])` and toast `"Added: {file.name}"`. Skip `disappeared` mutations.
  - If `FileSystemObserver` missing: emit toast "Watch folder is one-shot in this browser (no live updates)" but still ingest the initial snapshot.
  - `stopWatching()`: calls `observer.disconnect()` if active; clears stored handle + observer ref; emits toast "Stopped watching {dirName}".
- `src/stores/runtime.ts` — add `watchedFolderAtom: { handle, name, observer } | null` to track active watch state. Used by the hook for cleanup and by Toolbar for future "Stop watching" affordance (out of scope this task — just the atom plumbing).
- `src/hooks/useIngest.ts` — **export** the existing private `isAccepted` helper (and `getExt` if needed) so `useWatchFolder` can filter directory entries without duplicating the extension table.
- `src/components/shell/Toolbar.tsx` — remove `addWatchFolder` from the `@/stores/files` import; add `useWatchFolder` import; replace `addWatchFolder()` call in the "Watch folder" menu item with `startWatching()`. Keep the `setOpen(null)` trailing call.
- `src/stores/files.ts` — delete the `addWatchFolder` empty stub at line 86 (Phase 11/12 retirement precedent).
- `src/tests/watch-folder.test.ts` — unit tests for `pickDirectory` feature-detection (mock `window.showDirectoryPicker`); `isAccepted` is reused (already covered).
- `src/tests/watch-folder.spec.ts` — Playwright e2e: mock `window.showDirectoryPicker` to return a synthetic `FileSystemDirectoryHandle` with 3 image entries; trigger Toolbar "Watch folder"; assert 3 files appear in the queue + toast surfaces. Test 2: mock `'FileSystemObserver' in window === false` and assert the no-observer toast fires while the snapshot still ingests.

### Out of scope (parked for follow-up)
- "Stop watching" UI affordance (Toolbar item, StatusBar pill) — atom plumbing exists; UI lands next.
- Persisting `FileSystemDirectoryHandle` across reloads via IndexedDB (`structuredClone` of the handle). Requires permissions re-request flow; deferred.
- Recursive directory traversal (sub-folders) — single level for now per simplicity.
- Watching multiple folders simultaneously — single active watcher only.
- IDB persistence + permission re-prompt on next session.

<task id="4">
  <action>
    Create `src/hooks/useWatchFolder.ts`. Hook shape mirroring `src/hooks/useExport.ts` / `src/hooks/useSnippets.ts`. Returns `{ startWatching, stopWatching, isWatching }` where `isWatching` is `useStore(watchedFolderAtom) !== null`.

    `startWatching()` implementation:
    1. Call `stopWatching()` first (cleanup any existing watcher).
    2. `const handle = await pickDirectory()`. If `null`, return.
    3. Iterate `handle.values()` async; for each `entry` where `entry.kind === 'file'` AND `isAccepted({ name: entry.name } as File)`: call `await entry.getFile()` and push to `files: File[]`.
    4. Call `useIngest().ingest(files)`.
    5. Emit `pushToast({ kind: 'success', message: \`Watching \${handle.name} (\${files.length} files imported)\` })`.
    6. If `'FileSystemObserver' in window`:
        - `const observer = new window.FileSystemObserver(async (records) => { for (const r of records) { if ((r.type === 'appeared' || r.type === 'modified') && r.changedHandle?.kind === 'file' && isAccepted({ name: r.changedHandle.name } as File)) { const f = await r.changedHandle.getFile(); await useIngest().ingest([f]); pushToast({ kind: 'info', message: \`Added: \${f.name}\` }); } } })`
        - `await observer.observe(handle, { recursive: false })`
        - Store in `watchedFolderAtom.set({ name: handle.name, handle, observer })`.
    7. Else (no observer support):
        - `pushToast({ kind: 'info', message: 'Watch folder is one-shot in this browser (no live updates)' })`
        - `watchedFolderAtom.set({ name: handle.name, handle, observer: null })`.

    `stopWatching()` implementation:
    1. `const current = watchedFolderAtom.get()`. If `null`, return.
    2. If `current.observer`, call `current.observer.disconnect()`.
    3. `watchedFolderAtom.set(null)`.
    4. `pushToast({ kind: 'info', message: \`Stopped watching \${current.name}\` })`.

    Use `useIngest()` inside hook body. Use `filesAtom.get()`-style direct reads inside async bodies if any cross-atom access needed (matches project pattern from useExport/useSnippets).
