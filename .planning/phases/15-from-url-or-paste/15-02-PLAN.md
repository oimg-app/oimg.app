<objective>
Create the shared clipboard dispatcher that both surfaces (Toolbar onClick in 15-04 and document paste handler in 15-03) call. Two entry points: `pickFromClipboard(d)` for programmatic reads (Toolbar path) and `processClipboardEvent(e, d)` for synchronous `ClipboardEvent` walks (paste handler path). Single source of truth for the image-first → URL-string → no-op decision tree.

Purpose: SC-1 (clipboard image / URL ingest with success toast) and SC-3 (non-image clipboard surfaces the exact "no image or image URL" toast). Threat T-15-02 carried forward (DOMPurify in the existing useIngest pipeline already neutralizes malicious SVG bytes from the clipboard).
Output: `src/lib/clipboard-ingest.ts` + Node unit test + read-direction clipboard mocks.
</objective>

export function pickFromUrl(url: string): Promise<File | null>
