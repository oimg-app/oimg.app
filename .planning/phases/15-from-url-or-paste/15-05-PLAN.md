# Plan 15-05 — Gap closure

## Objective

Close the two user-surfaced gaps from `15-UAT.md`:

- **G-15-01** — Both ingest surfaces silently no-op on real-world inputs. Toolbar `From URL or paste` produces zero feedback (Popover unmount races sonner's render). Cmd/Ctrl+V of an image URL without a file extension misses the extension-only `IMAGE_URL_RE` gate and returns silently. Clipboard-read permission denials land on a bland "no image or image URL" message with no recovery path.
- **G-15-02** — Success toast waits for `await dispatcher.ingest(...)` which awaits the entire worker-pool optimize pipeline. Users see no feedback for seconds. Fix is structural: toast acknowledges *accept*, the existing file-row status indicators surface optimize progress.

Both gaps were diagnosed during `/gsd:verify-work 15`. Reproduction: Playwright on the running dev server confirmed `toast.*` IS called from the Toolbar path but the `<li>` never renders (G-15-01 race); user manually confirmed `Cmd+V` image-bytes path ingests correctly but feedback is late (G-15-02).

## Threat model

No new threats. Fix sits inside existing T-15-02 / T-15-01 / T-15-04 surfaces:
- The broadened URL check still funnels every fetch through `pickFromUrl` (T-15-01 `credentials: 'omit'` + Content-Type validation + size cap).
- Filename derivation still passes through `sanitizeBaseName` (T-15-04).
- The permission probe is read-only (`navigator.permissions.query`) — no side effects.
