<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| main thread → worker (Comlink) | EncodeJob (codec enum, sourceFormat, raw bytes, settings) crosses into the WASM worker |
| raw image bytes → jSquash WASM heap | Untrusted/possibly-malformed image buffer fed to native decoders |
| SVG bytes → svgo | Untrusted SVG text transformed (not executed) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-9-V5 | Denial of Service | codec.worker.ts (all cases) | mitigate | WR-02: `if (job.buffer.byteLength === 0) throw 'Empty buffer'` at top of every case before any await import — verified by -g "error" test |
| T-9-ENUM | Tampering | codec.worker.ts dispatch | mitigate | Keep existing `KNOWN_CODECS.has(String(job.codec))` guard before the switch — never index/import on raw codec value |
| T-9-SRC | Tampering | decodeSource | mitigate | decodeSource switches on a closed `sourceFormat` set and throws on default — unknown formats cannot reach a decoder |
| T-9-HEAP | Tampering | jSquash decoders | mitigate | Malformed buffer handled inside WASM; worker isolates WASM heap from main thread; outer try/catch rejects the job Promise without crashing the worker |
| T-9-SVG | Tampering | svgo SVG case | mitigate | svgo is a text transform — does not execute SVG/eval; no script execution path; DOMPurify available app-side if rendering raw SVG later |
| T-9-AVIF | (Robustness, not security) | AVIF case | accept | Safari <16.4 BigInt failure caught → rethrown descriptive; Plan 03 converts to per-file fallback (D-13). Not a security issue |
</threat_model>
