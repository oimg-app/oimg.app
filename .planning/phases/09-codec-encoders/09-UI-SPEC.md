# Phase 9 — UI Design Contract: Codec Encoders

> Visual and interaction contract for Phase 9. Generated by gsd-ui-researcher.
> SCOPE: This phase wires REAL encoded bytes to the EXISTING UI. The vast majority of UI
> components (CodecPanel, SvgoPanel, OutputPanel, CompareStage, DeltaStrip, file list,
> inspector) are pre-built. This contract covers ONLY the new or changed interaction/visual
> states Phase 9 introduces. Do NOT redesign or re-layout any pre-existing surface.

## New / Changed UI Surfaces — Detailed Contract

### 1. Inspector Context Label (D-03)

**What:** A single line directly above the CodecPanel section stack (inside the inspector pane header or as the first element of the inspector scroll area) that communicates "you are editing THIS file's settings."

**Placement:** Top of the inspector pane's scrollable content area, below the pane tab bar ("Codec | Output"), above the first `<Section>` component.

**When shown:**
- A file is selected: show the file name.
- No file selected: show the global-defaults message.

**Visual spec:**

```
[file-icon-dot 6px]  filename.png  [·  currently editing]
```

- Container: `flex items-center gap-2 px-3 py-1.5 bg-[var(--color-bg-2)] border-b border-[var(--color-line)]`
- Filename: `font-mono text-[11px] font-semibold text-[var(--color-fg-0)] truncate`
- "currently editing" suffix: `font-mono text-[10px] text-[var(--color-fg-3)]`
- File-type dot: 6px circle, `bg-[var(--color-accent)]` (colored if selected file; `bg-[var(--color-fg-3)]` if no file selected)
- No-file state: `font-mono text-[11px] text-[var(--color-fg-2)]` — copy: "Global defaults"
- Transition: none (selection changes are instant; no animation needed)

**Accessibility:** `aria-live="polite"` on the label container so screen readers announce file switches.

---

### 2. "Apply to All" Button (D-02)

**What:** A button that pushes the CURRENT GLOBAL DEFAULTS (settingsAtom) onto every file in the batch, triggering a batch re-encode queue. This is the primary new CTA of Phase 9.

**Placement:** Inside the inspector pane, below the inspector context label and above the first codec Section. Full-width within the inspector scroll area. NOT inside a Section component — it sits between the context label and the section stack.

**Visual spec:**

```
[  Apply global settings to all files  ]
```

- Container: `px-3 py-2`
- Button: `w-full h-7 rounded-[4px] bg-[var(--color-accent-dim)] hover:bg-[var(--color-accent)] text-[var(--color-accent)] hover:text-[var(--color-accent-fg)] font-mono text-[11px] font-semibold transition-colors cursor-default`
- Label: "Apply to all files" (6 words max — fits in 340px inspector width at 11px mono)
- Sub-label (below button, not inside): `font-mono text-[10px] text-[var(--color-fg-3)] px-3 pb-2` — copy: "Overwrites per-file settings"

**When visible:**
- Only when a file is selected AND the inspector is showing a per-file settings view (i.e., selectedFile is not null).
- When no file is selected (global defaults view): button label changes to "Set as global defaults" — NOT shown in Phase 9 (settingsAtom is already the global defaults source; "Set as global defaults" is implicit from editing the unselected state). RECOMMENDATION: Show "Apply to all files" always when at least 2 files exist in the queue. If only 1 file, hide the button (apply-to-all is meaningless with 1 file).

**Interaction:**
- Click → calls `applyToAll()` → triggers re-encode for all files → sonner toast: "Applying settings to {N} files…"
- Button does NOT show a confirmation dialog (non-destructive operation; files can be re-encoded again).
- No loading state on the button itself — the WorkerPool's BackpressureIndicator (pre-existing) communicates overall batch progress.

**Accessibility:** `type="button"`, `aria-label="Apply global codec settings to all files in queue"`.

---

### 3. Per-File vs Global Settings Distinction (D-02)

**What:** When a file is selected, the inspector shows that file's settings. When no file is selected (or the user is in "global defaults" mode), the inspector shows the global defaults.

**Context label differentiates the states** (see #1 above). No additional visual chrome needed beyond the context label.

**Global defaults mode indicator:**
- Context label reads: `"Global defaults"` in `--color-fg-2`
- No "currently editing" suffix.
- "Apply to all" button may be shown (see above).
- Inspector controls are bound to `settingsAtom` (existing behavior — no change to control wiring in global-defaults mode).

**Per-file mode indicator:**
- Context label reads: `"{filename}"  currently editing` in `--color-fg-0` + `--color-fg-3`.
- Inspector controls are bound to `selectedFile.settings` (new wiring from D-01/D-03).

No toggle, modal, or confirmation is needed. The context label IS the disambiguation surface.

---

### 4. Live Re-Encode In-Flight Indicator (D-05, D-07)

**What:** While the debounced re-encode is pending or running (slider being dragged, debounce timer active, or worker job in flight), the DeltaStrip must signal "results pending" so users understand why the numbers haven't updated.

**Affected component:** `DeltaStrip` — specifically the "OPTIMIZED" and "SAVED" DeltaCard cells.

**In-flight states:**

| State | Trigger | Visual |
|-------|---------|--------|
| Idle (stable) | No pending re-encode | Current DeltaStrip values, normal display |
| Encoding | Slider dragged or debounce timer active | Shimmer animation on OPTIMIZED + SAVED cells |
| Error | Encode failed (D-13) | Error values shown (see §5) |

**Shimmer spec (encoding state):**
- Apply to the OPTIMIZED and SAVED DeltaCard cells only (not ORIGINAL, SSIM, BUTTERAUGLI, DECODE).
- Shimmer: a moving gradient overlay on the cell value text — `animate-pulse` (Tailwind built-in, 2s ease-in-out infinite) on the value `<span>`.
- Value text during shimmer: replace with `"···"` (three centered dots) in `--color-fg-3` to clearly signal "computing" without layout shift.
- Sub-label during shimmer: keep showing codec/quality string (e.g., `WebP q82 m4`) so the user sees what they're encoding to.
- Shimmer color: `--color-fg-3` pulsing (not `--color-warn` — this is not a warning, just a loading state).
- Minimum shimmer duration: show for at least 300ms even if the encode completes faster (prevents flash of loading state for cached warm encodes).

**Implementation guidance (not design):** The `useLiveEncode` hook exposes an `encoding: boolean` state. CodecPanel passes this down to DeltaStrip via a prop or store key (recommend adding `encodingFileId: string | null` to `runtimeAtom` so DeltaStrip can subscribe without prop drilling).

### 5. Per-File Error State (D-13)

**What:** When an encode fails (codec error, unsupported format like AVIF on Safari <16.4), the file is marked with a per-file error state. The original bytes are kept as fallback output. The file list row shows an error indicator. A sonner toast surfaces the error.

#### 5a. File List Row Error Badge

**Placement:** Right side of the file list row, replacing the savings-percent badge when error state is present.

**Visual spec:**
- 8px filled circle: `w-2 h-2 rounded-full bg-[var(--color-err)] shrink-0`
- Left border stripe on the row: `border-l-2 border-[var(--color-err)]` (replacing the default `border-l-2 border-transparent` that selected/unselected rows use)
- Row filename: `text-[var(--color-fg-1)]` (no strikethrough — file still has fallback output)
- Row sub-label (file size): show original size with suffix `" · using original"` in `--color-fg-3`
- Tooltip (on hover of the error dot): show the error message string from `FileEntry.error`. Use `@radix-ui/react-tooltip` (already installed).

**Distinguishing errored-but-fallback from successfully-optimized:**
- Optimized file: shows savings-percent badge in `--color-accent` (green)
- Errored file: shows error dot in `--color-err` (red) + left border stripe in `--color-err`
- Errored file's "optimized" size in DeltaStrip OPTIMIZED cell: shows original size (since fallback = original bytes) with `"(fallback)"` appended in `--color-fg-3`

**Accessibility:**
- The error dot: `aria-hidden="true"` (decorative — error text in tooltip carries the message)
- Row: `aria-label="{filename} — encode failed, using original"` when error state present

#### 5b. Sonner Toast

**Trigger:** On each encode failure (one toast per file failure, not batched).

**Spec:**

| Property | Value |
|----------|-------|
| Variant | `toast.error(...)` (sonner built-in error variant — red icon) |
| Title | `"Encode failed: {filename}"` |
| Description | See copy table below |
| Duration | 6000ms (longer than default 4000ms — gives developer time to read) |
| Dismissible | Yes (sonner default) |
| Action | None (no retry button in Phase 9 — manual re-trigger by adjusting settings) |

**Toast copy by error type:**

| Error | Title | Description |
|-------|-------|-------------|
| AVIF on Safari <16.4 | `"Encode failed: {filename}"` | `"AVIF is not supported in this browser. Original file kept."` |
| Generic encode error | `"Encode failed: {filename}"` | `"Could not encode to {codec}. Original file kept. Try a different format."` |
| Empty/corrupt buffer | `"Encode failed: {filename}"` | `"File appears to be empty or corrupt. Check the source file."` |
| svgo parse error | `"Encode failed: {filename}"` | `"SVG could not be parsed. Original file kept."` |

---

### 6. Resize Controls States (D-10)

Pre-existing resize controls in `CodecPanel` (Resize section with `resizeOn` Switch + w/h Inputs + Fit/Algorithm SegControls) are wired to real encode this phase.

**No new UI states required.** The existing disabled/enabled states of the Resize section are sufficient.

**One flag:** The Quality slider in the CodecPanel parameters section must be visually disabled when codec is PNG (OxiPNG is lossless; quality has no effect). Currently the slider renders but has no effect.

**PNG quality slider spec:**
- When `settings.codec === 'PNG'`: Quality Slider receives `disabled` prop → renders with `opacity-50 pointer-events-none` (Radix Slider built-in disabled state).
- Quality label: add `" (lossless)"` suffix in `--color-fg-3` when PNG is selected: `"Quality (lossless)"`.
- Effort slider remains active for PNG (maps to OxiPNG `level` 0–6).

**JPEG progressive toggle (new — from 09-RESEARCH.md Open Question #1):**
Recommendation: Add a `progressive` boolean row to the JPEG parameters section in CodecPanel.
- Row: `grid grid-cols-[100px_1fr] gap-2.5 mb-2 items-center`
- Label: `"Progressive"` at `text-[12px] text-[var(--color-fg-2)]`
- Control: `<Switch checked={settings.progressive ?? true} onCheckedChange={setProgressive} />`
- Shown only when `settings.codec === 'JPEG'` (mirrors existing Lossless switch pattern).
- Default: `true` (progressive ON by default per jSquash meta.js).
