<objective>
Wire the real-bytes optimize path (D-04) and the debounced live re-encode (D-05/D-07). Replace `useOptimize`'s 0-byte stub jobs with real `File.arrayBuffer()` bytes carried via `rawBuffer`, dispatch through the pool with `Comlink.transfer`, and route results to `setFileResult` / failures to `setFileError` + a sonner toast (D-13 — keep original bytes as fallback). Create `useLiveEncode`, a debounced single-file trigger the inspector calls when the selected file's settings change. Add an `encodingFileId` to `runtimeAtom` so DeltaStrip can render the in-flight shimmer.

Purpose: This is the seam between the store (per-file settings + rawBuffer) and the encode engine (Plan 02 adapters). It is also where D-13's per-file error/fallback behavior is enforced at the application layer. The inspector (Plan 04) depends on `useLiveEncode` existing and on `runtimeAtom.encodingFileId`.
Output: A real `useOptimize`, a new `useLiveEncode` hook, and an `encodingFileId`/`setEncodingFile` addition to runtime.ts — all transferring buffers correctly so `rawBuffer` survives for repeated re-encodes.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| File handle → rawBuffer | User-dropped file bytes read into memory and cached on the FileEntry |
| hook → worker pool (Comlink) | EncodeJob carries raw bytes + per-file settings across the worker boundary |
| worker rejection → UI | Encode failure surfaces as per-file error state + toast (must not crash the batch) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-9-V5 | Denial of Service | useOptimize/useLiveEncode | mitigate | Entries with no rawBuffer are skipped (never dispatch 0-byte buffers); worker's WR-02 guard is the second line; Promise.allSettled isolates per-file failures so one bad file never aborts the batch |
| T-9-FB | (Robustness) | D-13 error path | mitigate | catch → setFileError + toast.error + retain original bytes as fallback output; other files continue (D-13) — batch always completes |
| T-9-XFER | Tampering | Comlink.transfer dispatch | mitigate | Always transfer a `rawBuffer.slice(0)` copy (Pitfall 3) so the cached original is never neutered; prevents empty-buffer dispatch on the next re-encode |
| T-9-PII | Information Disclosure | rawBuffer in memory | accept | Zero-server/zero-telemetry (CLAUDE.md) — bytes never leave the browser; in-memory cache is the intended design |
</threat_model>
