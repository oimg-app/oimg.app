## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| inspector control → store action | User input writes into the selected FileEntry.settings |
| encoded bytes → object URL → <img> | Encoded ArrayBuffer rendered as an image in the DOM |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-9-UI1 | Tampering | CodecPanel/SvgoPanel | mitigate | Components only call typed `setFileSettings<K extends keyof FileSettings>` — cannot write arbitrary settings keys; values flow through Plan 02's worker enum/guard before any WASM use |
| T-9-URL | Information Disclosure / Resource | CompareStage object URLs | mitigate | `URL.revokeObjectURL` in effect cleanup prevents blob-URL leaks; URLs are in-page only, zero-server (CLAUDE.md) — no exfiltration path |
| T-9-SVGR | Tampering | CompareStage SVG render | accept | SVG layer rendered as `<img src=blob>` (not inline DOM) — browsers do not execute scripts in img-loaded SVG; DOMPurify available if inline rendering is ever added (out of scope this phase) |
| T-9-A11Y | (Accessibility, not security) | InspectorPane/file row | mitigate | aria-live on context label, aria-label on Apply-to-all + errored rows per UI-SPEC; decorative dots aria-hidden |
</threat_model>
