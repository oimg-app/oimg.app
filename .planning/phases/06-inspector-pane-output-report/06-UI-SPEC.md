## Interaction Contract

### OutputPanel (INSP-07)

- Panel renders when `uiAtom.tab === 'output'`
- When `$selectedFile` is null: render empty state (heading + body text, no sections)
- When `$selectedFile` is set: render 3 `Section` blocks in order: Base64, URL-encoded, `<picture>`
- Each section contains:
  - A `<pre>` or `<code>` block with the snippet text (font-mono, 12px, bg-2 background, rounded-md, px-3 py-2, overflow-x-auto)
  - A "Copy" `Button` variant="ghost" size="sm" with `Copy` phosphor icon at left
  - On click: call `navigator.clipboard.writeText(snippetText)` — on success flash button label to "Copied!" for 1500ms; on failure call `pushToast` with error copy
- Snippet content is stubbed from `$selectedFile` data (name, type, dimensions)
- No external network calls — zero-server constraint maintained

### ReportPanel (INSP-08)

- Panel renders when `uiAtom.tab === 'report'`
- When `filesAtom.entries` is empty: render empty state (heading + body text)
- "Total savings" section:
  - Stats grid: 4 cells in a 2×2 grid — Before (orig bytes total), After (opt bytes total), Saved (−bytes), Files (count) — values from `$totals`
  - Per-file bar chart: horizontal row of bars, one per `filesAtom.entries` item
    - Bar height represents savings % (0–100% maps to 0–48px max height)
    - Bar minimum rendered height: 4px (visual floor)
    - Bar color: `accent` if savings ≥ 30%; `warn` if savings < 30%
    - Bar tooltip on hover: `{file.name} · {savings%}` formatted via `fmtPct`
    - Bars are not interactive (no click action in this phase)
- "Format breakdown" section:
  - One row per unique format present in `filesAtom.entries`
  - Each row: format type label (e.g. "WebP") | file count | bytes saved
  - Rows separated by `Separator` component
  - Format label uses format badge color convention from FileRow (SVG=purple, PNG=blue, WebP=cyan, JPEG=orange, AVIF=rose)

### Tab integration (from Phase 4 InspectorPane)

- Tab auto-switch logic lives in `InspectorPane` (Phase 4, INSP-01) — Phase 6 does not modify it
- OutputPanel and ReportPanel are slot-mounted; InspectorPane controls visibility via `uiAtom.tab`

## Layout

OutputPanel and ReportPanel live inside `InspectorPane` right pane (~260–340px width via `--width-tweaks-panel`). Both panels:
- Fill available height with `overflow-y-auto`
- Use `Section` component exclusively (no custom section wrappers)
- No horizontal scroll except inside `<pre>` snippet blocks
