## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| component → clipboard API | navigator.clipboard.writeText is a browser-permissioned sink; failure path must be handled |
| stub data → rendered snippet | FileEntry fields (name, dim, target) flow into a string injected into a `<pre>` text node |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information disclosure | clipboard write | accept | Snippet contains only stub/derived strings from local FileEntry; no PII, no network, zero-server constraint holds |
| T-06-02 | Tampering/XSS | snippet text in `<pre>` | mitigate | Render snippet as auto-escaped React text child; never use raw-HTML injection props; builders return plain strings |
| T-06-03 | Denial of clipboard | writeText rejects (permissions) | mitigate | try/catch around writeText; on reject call pushToast with the UI-SPEC error copy, never throw |
</threat_model>
