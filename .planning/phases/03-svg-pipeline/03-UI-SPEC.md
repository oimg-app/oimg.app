# Phase 3 ‚Äî UI Design Contract

> Visual and interaction contract for the SVG pipeline. Generated by gsd-ui-researcher; verified by gsd-ui-checker. All visual primitives are inherited verbatim from Phase 1 (`example-ui/OIMG.html` port). Phase 3 adds **no** new tokens, **no** new typography scales, **no** new component primitives ‚Äî only behavior + copy.

## Color

Phase 3 reuses the Phase 1 oklch palette verbatim. The 60/30/10 split is already established by the example-ui port.

| Role | Token | Light value | Dark value | Usage in Phase 3 |
|------|-------|-------------|------------|------------------|
| Dominant (60%) | `--bg-0` / `--bg-1` | `oklch(0.985 0.003 250)` / `oklch(0.965 0.004 250)` | `oklch(0.165 0.008 250)` / `oklch(0.205 0.009 250)` | Inspector pane body, snippet code blocks, plugin row hover |
| Secondary (30%) | `--bg-2` / `--bg-3` | `oklch(0.945 ‚Ä¶)` / `oklch(0.915 ‚Ä¶)` | `oklch(0.235 ‚Ä¶)` / `oklch(0.275 ‚Ä¶)` | `Section` badge backgrounds, `Toggle` track, `.copy-btn` background, `Tooltip` trigger hover |
| Accent (10%) | `--accent` | `oklch(0.62 0.18 145)` | `oklch(0.80 0.17 145)` | See "Accent reserved for" below |
| Warn | `--warn` | `oklch(0.62 0.16 65)` | `oklch(0.82 0.15 75)` | Sanitization "removed N elements" badge on file row; foot-gun warning hint icon glyph color (text stays `--fg-3`) |
| Err | `--err` | `oklch(0.58 0.20 25)` | `oklch(0.72 0.19 25)` | Clipboard-copy failure toast (sonner default error styling); SVGO/sanitizer hard-fail file status dot |
| Info | `--info` | `oklch(0.55 0.15 235)` | `oklch(0.78 0.13 235)` | Re-optimize-in-flight pulse on the inspected file's status dot (existing `processing` dot ‚Äî already wired) |

**Accent reserved for (explicit list ‚Äî Phase 3 must NOT broaden this):**
- Plugin checkbox `on` state (filled square, `--accent`)
- Plugin row "saves" % text when value > 0 (`var(--accent)` per existing `.plugin .saves` rule)
- `Toggle` `on` state for the global "Disable SVG sanitization on export" switch
- `Section` `badge.acc` modifier ‚Äî used on the SVGO preset badge (`preset-default`) and snippet type badges (`inline`, `data-uri`)
- `.copy-btn.ok` (post-copy success state, 1100ms) ‚Äî existing pattern preserved
- File status dot `done` (already wired Phase 2; not new)
- Selected file row left-edge stripe (`.file-row.selected::before`; not new)

**Warn reserved for (Phase 3 introduces):**
- File-row "sanitized" badge ‚Äî small text/pill, color `var(--warn)`, background `oklch(from var(--warn) l c h / 0.16)` (matches existing `.pill.warn` rule). Renders ONLY when the SVG adapter reports `removed.length > 0`.
- Foot-gun warning glyph (info-circle icon) inline before the hint text on `removeViewBox` / `removeDimensions` / `cleanupIds` rows. Glyph stroke = `var(--warn)`; hint body text stays `var(--fg-3)` (existing aggressive-mode hint copy color) ‚Äî keeps the row visually quiet while drawing the eye.

**Destructive (`--err`) reserved for:**
- Sonner toast on clipboard-copy failure (`toast.error(...)` pre-existing pattern, preserved in `SnippetPanel`).
- File status dot when SVGO crashes or DOMPurify rejects malformed SVG (`AdapterError`).

**Constraint:** Phase 3 must NOT introduce additional accent uses beyond the list above. The user's choice for D-03 (per-file badge, no toast) explicitly trades batch-level accent noise for a quieter warn-tier signal.

### SvgoPanel ‚Äî Foot-gun hint copy (verbatim)

Hints render as `<small>` text directly below the affected `.plugin` row, prefixed by a 13√ó13 info-circle glyph in `var(--warn)`. Hint container reuses the existing aggressive-mode paragraph style: `font: 10.5px var(--mono); color: var(--fg-3); line-height: 1.5; margin: 4px 0 0`.

| Plugin | Hint copy |
|--------|-----------|
| `cleanupIds` | `May break external CSS or `<use>` references that target SVG element ids` |
| `removeViewBox` | `Disabling viewBox can break responsive scaling when the SVG is embedded in HTML or CSS` |
| `removeDimensions` | `Removes width/height attributes ‚Äî only safe when viewBox is preserved` |

**Affordance choice (resolves Claude's discretion):** Inline `<small>` with leading info glyph. Rejected: hover-only tooltip (hides the warning until interaction), info icon with click-to-expand (extra friction on a settings panel). Inline always-visible matches the user's locked decision "show a warning when toggling foot-gun plugins" ‚Äî _showing_ is the contract.

**When to display:** Always visible (not gated on toggle state). The warning is preventative ‚Äî users need to see it BEFORE flipping a foot-gun. Foot-gun rows therefore have a fixed 2-line height (row + hint) regardless of toggle state.

### Global sanitization toggle (D-04)

**Toggle location (resolves Claude's discretion):** A new `Section` titled `Sanitization` is appended to `SvgoPanel` (BELOW the Plugins section, above the section border). **Rejected** alternatives:

- TweaksPanel "Global" subsection ‚Äî there is no `TweaksPanel.tsx` in the current codebase (verified). The brief's "TweaksPanel" refers to the conceptual right-pane settings family. Adding a new top-level panel for one toggle is overkill.
- Hidden behind an "Advanced" expandable ‚Äî defeats the user's explicit choice in D-04 ("clearly labeled as advanced"). The Section's body copy carries the "advanced" framing without burying the control.

**Section copy (verbatim):**

| Element | Copy |
|---------|------|
| Section title | `Sanitization` |
| Section badge | `safe` (when toggle off, badge.acc modifier) / `unsafe` (when toggle on, the existing `.pill.warn` rule applies via badge custom class) |
| Toggle row label | `Disable on export` |
| Toggle row helper text | `Advanced. Skips the DOMPurify pass on the exported SVG. Preview, snippets, and ZIP all use the unsanitized output. Off by default.` |

The Toggle row reuses the same flex layout as the (now-deleted) Aggressive mode row: label on the left at `11.5px var(--fg-2)`, `Toggle` component on the right, `<small>` helper text below at `10.5px var(--mono) var(--fg-3) line-height: 1.5`.

### SnippetPanel ‚Äî Section structure and copy

`OutputPanel` is replaced by `SnippetPanel` (D-12). For SVG files, exactly two snippet sections render in this order:

| # | Section title | Section badge | code-row label | Snippet content |
|---|---------------|---------------|----------------|-----------------|
| 1 | `Inline SVG` | `inline` (badge.acc) | `<svg>` | The optimized + sanitized SVG markup, verbatim, in a `<pre class="code">` block. `max-height: 200px` per existing OutputPanel pattern; scrolls when long. |
| 2 | `Data URI ¬∑ URL-encoded` | `data-uri` (badge.acc) | `CSS background` | `url("data:image/svg+xml,${yokselEncoded}")` in a `<pre class="code">` block. `max-height: 140px` (existing `.code` default). |

**Per-snippet enable/disable checkbox (D-13):** Each `Section` header gets a checkbox to its left of the title (16√ó16, reuses the existing `.popover .pi.check` glyph rule). When unchecked, the section body collapses to a single muted row reading `Disabled. Enable above to include in copy-all output.` ‚Äî body copy color `var(--fg-3)`, 11.5px sans. Checkbox state lives in `useSettingsStore.snippetTogglesByFileId[fileId][snippetId]: boolean`, default `true`.

**Per-snippet copy button:** Existing `.copy-btn` pattern preserved verbatim from `OutputPanel`. Button right-aligned in `.code-row`. Label cycles `copy` ‚Üí `copied` (1100ms) ‚Üí `copy`. On clipboard failure, sonner error toast shows `Copy failed` (existing copy preserved). `Clipboard unavailable` (existing copy preserved) shown when `navigator.clipboard?.writeText` is undefined.

**Non-SVG snippet rows:** The snippet registry includes stubs for `picture`, `img-srcset`, `data-uri-base64`, `inline-img-svg`. Their `applicableFormats` excludes `svg`, so they render NOTHING when an SVG file is inspected. (Phase 6 plugs in raster generators; Phase 3 ships the stubs to lock the registry shape.)

**Empty / no-data states:**

| Condition | Copy / behavior |
|-----------|----------------|
| File has not yet been optimized (status `queued` or `processing`) | Both snippet sections render the section frame + checkbox + copy-row, but the `<pre class="code">` block reads `// Run Optimize to generate snippet` in `var(--fg-3)`, italic. Copy button is disabled (`opacity: 0.5`, no hover state, click is no-op). |
| File errored during SVG processing | `<pre class="code">` reads `// Snippet unavailable ‚Äî see Report tab`, color `var(--fg-3)`. Copy button disabled. |
| Sanitization removed content | No special snippet-panel state (the badge lives on the file row, not the snippet panel). Snippet output is the sanitized markup as normal. |

### File-row sanitized badge (D-03)

**Badge shape (resolves Claude's discretion: count vs boolean):** Show the count. A boolean "sanitized" pill is a binary security signal; the count tells the user *how much* was stripped, which is the actionable information for trust calibration. Cost is one number in `AdapterMeta` ‚Äî already on the planning radar.

**Visual:** Inline element appended to `.file-stat` (after the existing arrow + savings spans). Reuses the existing `.pill.warn` color rule (`var(--warn)` text, `oklch(from var(--warn) l c h / 0.16)` background, transparent border). Height matches existing `.pill` (18px) but with smaller `0 5px` horizontal padding to fit the dense file-stat row.

**Copy (verbatim):**

| Sanitized count | Badge text |
|-----------------|------------|
| `0` (or `undefined`) | (badge not rendered) |
| `1` | `sanitized ¬∑ 1` |
| `n > 1` | `sanitized ¬∑ ${n}` |

**Tooltip on hover (existing `Tooltip` primitive):** `${n} dangerous element${n === 1 ? '' : 's'} removed by DOMPurify` ‚Äî explains the meaning without a toast. Tooltip placement: `top` (above the badge) so it doesn't push the row layout.

**Aria:** Badge is a `<span>` with `aria-label="${n} elements removed by sanitizer"`. ARIA live announcement on first appearance is **not** added ‚Äî users see the badge when they look at the file row, which matches the user's locked "no toast, badge only" decision.

### Toolbar / status interactions (no new copy)

- Existing `Optimize` button (Phase 2) drives the batch SVGO + DOMPurify pipeline. No copy change.
- Existing `Workers` pill (Phase 2) reflects the same in-flight count when SVGO preview jobs run on plugin toggle. No copy change.
- Existing ARIA live region announces `done` / `error` via `lib/live-region#announce` (Phase 2). For sanitized SVGs, the standard "done" announcement is sufficient; sanitization itself is intentionally non-vocal (D-03).
