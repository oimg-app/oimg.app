<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Snippet string → OS clipboard via navigator.clipboard.writeText | Text crosses the renderer→OS boundary; treated as opaque text bytes (no HTML interpretation at this layer) |
| Snippet string → hidden textarea + execCommand('copy') | Fallback path; textarea value is read by document.execCommand inside renderer, then handed to OS |
| Clipboard permission API → user | Permission prompt is browser-mediated; not under app control |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-03 | Denial of Service | navigator.clipboard.writeText throws in non-secure context OR permission-denied | mitigate | Task 1 feature-detects `window.isSecureContext === true` AND `'clipboard' in navigator` AND `typeof navigator.clipboard?.writeText === 'function'`; on rejection falls through to textarea+execCommand silently; on dual failure toasts 'Copy failed — try again' but never throws to caller |
| T-12-04 | Information Disclosure | execCommand('copy') deprecated path leaves hidden textarea in DOM on synchronous throw | mitigate | Task 1 wraps `document.execCommand('copy')` in try/catch/finally; finally block ALWAYS runs `document.body.removeChild(ta)` (acceptance criteria asserts the literal `finally` token is present in the file) |
| T-12-LOG | Information Disclosure | console.error of clipboard text leaking via DevTools / shipped logs | mitigate | Task 1 acceptance criteria asserts zero `console.log` / `console.error` calls in non-comment lines (zero-telemetry constraint from CLAUDE.md) |
</threat_model>
