## Phase Surfaces — Locked Specifications

This section extends the standard contract with phase-specific picks. Each entry has acceptance criteria the executor must satisfy.

### Surface 1 — File-row source-density control (Claude's discretion, locked)

**Pick: hover/focus-revealed inline `<select>`** (NOT a permanent always-visible dropdown).

**Rationale:** the existing file-row is already dense (28px thumb · 1fr name+stat · auto status-pill). Phase 3 added the `sanitized · N` badge inside the stat line. A permanent always-visible dropdown crowds the row beyond legibility on narrow widths (<320px panel). A hover/focus-revealed control matches the existing iconbtn pattern in pane headers and stays out of the way until needed.

**Visual contract:**
- File-row baseline (idle) shows: thumb · name · `@Nx` density tag in stat line · status pill. No dropdown chrome visible.
- On row `:hover` OR keyboard focus on row: an 18×18px `chevron-down` icon button appears at the right edge of the stat line (after the density tag), in `var(--fg-2)`.
- Click/Enter on the icon button: opens a Radix `Popover` anchored bottom-right, `var(--bg-2)` fill, `var(--line)` border, 4px radius, `var(--shadow-elev)` shadow. Popover contains a vertical 3-button segmented control (`1x` / `2x` / `3x`) using the existing `.seg-sm` style.
- Selecting an option: closes the popover, mutates `useFilesStore.setSourceDensity(sourceId, density)`, re-renders the file-row stat line with the new density tag, and triggers `addFile`-style fan-out (or in-place mutation per planner discretion).

**Keyboard:**
- Tab order: file-row → chevron button → status pill (existing). Chevron button has `aria-label="Change source density (currently {density})"`.
- Enter/Space opens popover. Arrow Up/Down moves between options. Enter/Space selects. Esc closes without committing.

**Mobile:** the chevron button is also visible on `:focus-within`; tap opens popover. No hover-only state.

**Acceptance:**
- [ ] Permanent file-row layout unchanged on idle.
- [ ] Chevron appears on row hover and persists when popover is open.
- [ ] Popover uses Radix Popover for focus management + Esc handling.
- [ ] Density-tag in stat line updates immediately on commit.
- [ ] WCAG AA: hit target ≥ 24×24 (button has 6px padding around 18px icon).

### Surface 2 — File-row target-density checkboxes

**Visual contract:**
- Three checkboxes, horizontal, mono labels `1x` `2x` `3x`, each 13×13px box + 4px gap + label.
- Group lives in the file-row stat line, after the `@Nx` density tag, before the chevron button. Total stat-line layout: `[@Nx tag] · [1x☑ 2x☑ 3x☐] · [chevron]`.
- The source-density checkbox is **auto-checked + LOCKED**:
  - `cursor: not-allowed` on the input + label.
  - Background fill `var(--accent-dim)` instead of solid `var(--accent)` (subtle distinction from a freely-checkable "on" state).
  - `aria-disabled="true"`, `disabled` attribute, and a Radix Tooltip on hover/focus reading: `Source density ({Nx}) — included automatically`.
- Other two checkboxes use full `var(--accent)` solid fill when checked.
- Empty target-set guard: if user attempts to uncheck the last remaining non-source checkbox (e.g., source is 2x, both 1x and 3x are off, user tries to uncheck 2x — blocked at source level — but if user ALSO unchecks the only non-source target, the system MUST keep at least the locked source). Inline error `Pick at least one density` shows above the checkbox row in `var(--err)`, 11.5px, for 2s, then auto-dismisses. The uncheck attempt is rejected (no state mutation).

**Keyboard:**
- Each checkbox is in tab order (after the chevron button of the previous row).
- Space toggles. Locked source-density checkbox skips toggle on Space (no state change) and announces `"Source density, locked"` via `aria-live` on first focus.

**Acceptance:**
- [ ] Source checkbox is visually distinct (dim accent, not solid).
- [ ] Source checkbox cannot be unchecked via mouse OR keyboard.
- [ ] Inline error fires + auto-dismisses on empty-target attempt.
- [ ] WCAG AA color contrast: ≥4.5:1 for label text against background in both themes (verified with the existing `var(--fg-0)` on `var(--bg-0)` token pair).

### Surface 3 — File-row variant grouping (Claude's discretion, locked)

**Pick: flat-with-density-tag, family-grouped via 2px left rail.**

**Why this pick over parent+nested or collapsed-by-default:**
- *Parent + nested children*: introduces a new disclosure-arrow control + accordion state, adds toggle-state UI complexity, and breaks the existing flat scroll/keyboard-nav model. Heavy.
- *Collapsed-by-default group*: hides downstream variants behind one click — surprises the user who just dragged in 50 PNGs and expects to see all 150 variants.
- *Flat-with-tag + 2px rail*: every variant appears as its own row (matches D-04+D-14's "each density variant is its own FileEntry" data model 1:1), the `@Nx` tag in the stat line tells you which density it is, and a 2px-wide vertical rail in `var(--accent-dim)` on the left edge of every variant row visually groups them with their siblings of the same source. Source identity is conveyed by name prefix (already there: `logo@1x.png`, `logo@2x.png`, `logo@3x.png`). No new disclosure controls. No new keyboard model.

**Visual contract:**
- All variants of a single source share an identical 2px-wide vertical bar on their left edge, color `var(--accent-dim)`, full row height. (Independent of the existing `selected::before` 2px solid `var(--accent)` rail for the selected file — the two never coincide because selection is per-row, not per-family.)
- Family identity is computed from the FileEntry id prefix (`{sourceUuid}-{Nx}` → group by `{sourceUuid}`). Each family gets a stable color slot from a deterministic palette of 6 dim-accent variants (cycling through `oklch(0.80 0.17 {145|55|235|320|25|175} / 0.16)`); same source = same hue.
- Variants display in 1x → 2x → 3x order regardless of upload order (sort key = density).
- The 2px family rail is rendered via `box-shadow: inset 2px 0 0 var(--family-color)` on each variant row to avoid disturbing the existing grid layout.

**Acceptance:**
- [ ] Variants of the same source share a colored left rail; different sources have visually distinct rails.
- [ ] Selection rail (existing 2px `var(--accent)`) overrides family rail on the selected row only.
- [ ] Family color is stable across page interactions (re-renders, sort, filter).
- [ ] No new disclosure widgets are introduced.
- [ ] Variants sort 1x → 2x → 3x within a family.

On SVG-active panel, this section still renders (resize affects rasters; SVG variants are a no-op but the global-config UX should be consistent).

**Acceptance:**
- [ ] Section renders second in the panel ordering.
- [ ] Algorithm dropdown defaults to `Lanczos3 (default)`.
- [ ] Selecting a different algorithm updates global state and re-runs in-flight previews (existing Phase 3 `enqueuePreview` debounce reused by raster).
- [ ] Section visible on all codec tabs (SVG + each raster).

### Surface 6 — StatusBar backpressure indicator (Claude's discretion, locked)

**Pick: text + dot pill — NOT icon-only and NOT progress-bar overlay.**

**Rationale:**
- *Icon + tooltip*: discoverable only on hover; users miss it on a glance.
- *Text badge*: reads cleanly in the existing StatusBar item-row pattern; mirrors how "5 workers running" already communicates state via dot-pip + text.
- *Progress-bar overlay*: invasive; competes with the worker-pip and is harder to discriminate at the StatusBar's 22px height.

**Visual contract:**
- New StatusBar item, inserted as the SECOND item (after the worker-pip, before "SVGO 4.0.1"). Hidden when `useRuntimeStore.throttleActive === false`.
- Active rendering: `<span class="item">` with two children:
  - 8px-square dot in `var(--accent)` (matches the worker-pip dimensions exactly), animated with a 1.4s ease-in-out pulse identical to the existing `.file-status.processing` pulse.
  - Text `Pacing` in 11.5px mono, color `var(--warn)`.
- Tooltip on hover/focus (Radix Tooltip): `Memory pacing active — admission gate is throttling new jobs to keep peak heap under 600MB.`
- Item disappears (via conditional render, NOT visibility:hidden — so it does not occupy layout space) when throttling ends.

**Lifecycle:**
- Shows when admission gate first throttles the batch.
- Persists for the rest of the batch even if intake unblocks (per CONTEXT.md D-13 wording: "for the rest of that batch, a small badge ... indicates active throttling").
- Clears at batch end (when last variant transitions to `done`/`error`).

**Acceptance:**
- [ ] Pill is invisible (zero DOM) until first throttle event.
- [ ] Once shown, persists for the rest of the batch.
- [ ] Disappears completely when batch finishes.
- [ ] Tooltip is keyboard-focusable; Esc dismisses.
- [ ] Pulse animation does NOT coincide-clash with worker-pip pulse (both use the same pulse keyframes — visually consistent).

### Surface 7 — First-throttle toast (Sonner)

**Locked:**
- Type: `toast.info(...)`.
- Title: `Pacing batch for memory`.
- Description: `Some files are queued briefly to keep the tab responsive.`
- Duration: Sonner default (4000ms).
- Fires: ONCE per batch, on the first admission-gate trigger. Subsequent throttle events in the same batch are silent (StatusBar pill carries the persistent signal).
- State home: `useRuntimeStore.firstThrottleFiredForBatch` (boolean, reset on batch start).

**Acceptance:**
- [ ] Toast fires exactly once per batch even if gate triggers 50 times.
- [ ] Toast does NOT fire when batch never throttles.
- [ ] Reset path: starting a new batch clears the flag.

### Surface 8 — Filename-collision toast (Sonner)

**Locked:**
- Type: `toast.info(...)`.
- Title: `{N} files renamed to avoid collisions`.
- Description: `Suffix "(2)", "(3)", … inserted before "@Nx" so each variant has a unique name.`
- Duration: Sonner default (4000ms).
- Fires: ONCE per batch (per `addFile` operation, not per individual collision). Aggregate count `N` reflects total renames in this batch.
- State home: `useRuntimeStore.collisionRenameCountForBatch` (number, reset on batch start). Toast fires when count goes from 0 to ≥1 in a single `addFile` invocation.

**Acceptance:**
- [ ] Single drop of 5 colliding files = 1 toast with `N=5`.
- [ ] Sequential drops accumulate count but only fire one toast per drop event.
- [ ] No toast when no collisions occur.
