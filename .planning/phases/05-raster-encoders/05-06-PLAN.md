The format routing in useBatchOrchestrate uses `f.format` (from the FileEntry), so adding a file with `format: 'jpeg'` will route it through jpeg-adapter regardless of the blob's actual bytes.

**Important caveat:** jpeg-adapter.ts calls `decode(input)` which tries to decode as JPEG. If the input bytes are a PNG, this will throw `AdapterError('jpeg', 'decode', ...)`. So for JPEG/WebP/AVIF tests, we need ACTUAL bytes of those formats.
