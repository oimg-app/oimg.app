## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| main thread → worker | `EncodeJob` (codec string + raw file ArrayBuffer + settings) crosses into the worker; buffer is untrusted file content |
| worker → WASM codec | raw buffer fed to jSquash decode; malformed bytes can throw inside WASM |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Denial of Service | codec.worker.ts optimize() on malformed buffer | mitigate | Wrap the codec switch in try/catch; jSquash rejection re-thrown as a rejected job Promise — the worker process never crashes, only the single job fails (RESEARCH.md Security Domain) |
| T-08-03 | Tampering | codec.worker.ts `job.codec` dispatch | mitigate | Validate `job.codec` against the literal 5-value enum before the switch; invalid → throw; no dynamic property indexing, no `eval`/`Function()` on `job.settings` (ASVS V5 Input Validation) |
| T-08-04 | Information Disclosure | initial-route bundle leaking AVIF WASM | mitigate | All `@jsquash/*` imports are `await import()` inside their switch branch — AVIF's ~8MB WASM is never in the initial chunk (PIPE-02); verified by Plan 01's worker-pipeline.spec.ts network assertion |
| T-08-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed (comlink + @jsquash/* pre-existing; RESEARCH.md Package Legitimacy Audit) — no install step, no slopcheck gate |
