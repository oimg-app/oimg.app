# Phase 14 Plan 02: src/sw.ts Workbox precache + CacheFirst codec wasm — Summary

## Decisions Made

1. **`url.pathname.endsWith('.wasm')` over `request.url.endsWith('.wasm')`** — pathname strips query strings, future-proofing against cache-busting suffix patterns Plan 14-04+ may add.
2. **Intersection-typed `declare const self`** — combines `ServiceWorkerGlobalScope` with `{ __WB_MANIFEST: Array<{url, revision}> }` in one declaration; avoids a separate ambient module shim.
3. **Bypass `npm run build`'s `tsc -b` gate via direct `./node_modules/.bin/vite build`** — pre-existing tsc errors (snippets.ts FileEntry import, toolbar-snippets.spec.ts defaultFileSettings, pwa.spec.ts Wave-0 RED stub `'serviceworker'` event-type literal) are documented as baseline RED (MEMORY `typecheck-and-test-gotchas`, 14-01-SUMMARY decision #4). vite build itself succeeded cleanly. Fixing the baseline is out of scope for this plan (Rule 1 SCOPE BOUNDARY).
4. **No COOP/COEP header injection in any route handler** (T-14-COEP) — RESEARCH §Critical Integration Risk: COOP/COEP confirms cached Response objects retain their original headers; SW must not shadow them.
