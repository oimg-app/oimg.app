<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Dev Server | Vite dev server serves responses; security headers injected here for local dev |
| Browser → Cloudflare Pages | CDN serves `_headers` file headers; production isolation boundary |
| npm → bundle | @fontsource-variable packages are third-party; bundled into dist at build time |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | Browser + all origins | mitigate | `Cross-Origin-Opener-Policy: same-origin` in vite.config.ts server.headers (dev) and public/_headers (prod) — isolates browsing context, prevents Spectre-class cross-window timing attacks |
| T-01-02 | Tampering | Browser + cross-origin resources | mitigate | `Cross-Origin-Embedder-Policy: require-corp` — blocks cross-origin sub-resources lacking CORP header; prevents COEP bypass via third-party CDN resources |
| T-01-03 | Information Disclosure | Google Fonts CDN | mitigate | Remove `@import url("https://fonts.googleapis.com/...")` from src/index.css; replace with @fontsource-variable npm packages (same-origin bundled) — eliminates COEP violation vector |
| T-01-04 | Tampering | public/_headers | accept | _headers file is static and committed to git; no runtime modification possible; Cloudflare Pages serves verbatim |
</threat_model>
