# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Shell + Foundation.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

| Property | Value |
|----------|-------|
| Tool | shadcn/ui |

| xl | 16px | Compare stage checkerboard cell size |
| 2xl | 24px | — |

### Token Mapping to src/index.css

The existing `src/index.css` uses a simplified 2-stop zero-chroma scale. The prototype requires a 5-stop scale with blue-grey chroma (~0.008–0.012 at hue 250). Before porting components, expand `src/index.css` to add `--color-bg-1`, `--color-bg-2`, `--color-bg-3`, `--color-line`, `--color-line-strong`, `--color-fg-0` through `--color-fg-3` mapping verbatim to the prototype values above. Map to shadcn semantic aliases where possible; add OIMG-specific variables for bg-N and fg-N that have no shadcn equivalent.

**Accent reserved for:** Primary CTA button (Download ZIP / Optimize), selected file-row left border and background tint, file-status dot (done), savings percentage label in file stat (positive), dropzone over-state border and background, split-handle bar, toggle knob (active state), focus ring (`--ring`), `<kbd>` accent text. **Never use accent for purely decorative elements or secondary buttons.**

**Destructive reserved for:** Error file-status dot, future destructive action buttons (remove file, clear all) only.

## Interaction Contract

### Theme Toggle
- Location: Title bar, top-right
- Trigger: Click sun/moon icon button
- Behavior: Toggle `.dark` class on `document.documentElement`; persist to `localStorage["oimg-theme"]`; default to `dark` if no preference stored
- Icon: Sun icon when dark (clicking switches to light), Moon icon when light (clicking switches to dark)
- Accessible label: `aria-label="Switch to light theme"` / `aria-label="Switch to dark theme"` (updates on toggle)

### File Row Selection
- Click: select file row; apply `background: var(--accent-dim)` + left 2px accent border
- Keyboard: Arrow up/down navigate list; Enter/Space select; Tab moves focus out of list
- ARIA: `role="listbox"` on `.filelist`, `role="option"` + `aria-selected` on each `.file-row`

### Drop Zone
- Idle: dashed border `var(--line-strong)`, muted text
- Drag over: `border-color: var(--accent)`, `background: var(--accent-dim)`, text `var(--fg-0)`
- ARIA: `aria-label="Drop image files here"`, `aria-dropeffect="copy"`

### Toolbar Buttons
- Default: `background: var(--bg-2)`, `border: 1px solid var(--line)`, height 28px
- Hover: `background: var(--bg-3)`, `border-color: var(--line-strong)`
- Primary (Download/Optimize): `background: var(--accent)`, `color: var(--accent-fg)`, no border
- Ghost: transparent background, hover shows `var(--bg-2)`
- Focus ring: `outline: 2px solid var(--ring)` (`--ring` = accent green), `outline-offset: 2px`
- Keyboard: all buttons reachable via Tab; activation via Enter or Space

### Accordion (TweaksPanel)
- Use shadcn `Accordion` (Radix-based)
- Default: Global settings section open; format sections collapsed
- Keyboard: Arrow keys navigate items; Enter/Space toggle; Home/End jump to first/last
- ARIA: `aria-expanded` managed by Radix

### Slider (TweaksPanel quality controls)
- Use shadcn `Slider` (Radix-based)
- Range: codec-dependent (e.g. JPEG quality 0–100, OxiPNG level 0–6)
- Keyboard: Arrow keys step by 1; Page Up/Down step by 10; Home/End jump to min/max
- Display: numeric value readout adjacent to slider, JetBrains Mono, tabular nums

### Split Handle (DetailPanel)
- Draggable vertical handle at `--split` position (default 50%)
- Width: 1px accent color line with 6px invisible grab zone each side (total 13px hit area)
- Cursor: `col-resize` on hover
- Keyboard: Left/Right arrows move split by 5% per keypress; range 5%–95%
- ARIA: `role="slider"` `aria-label="Compare split position"` `aria-valuemin="0"` `aria-valuemax="100"` `aria-valuenow="{pct}"`

### Status Processing Animation
- Processing files: status dot pulses `opacity: 0.4 → 1 → 0.4` at 1.4s cycle (`var(--info)` color)
- No animation for done/queued/error states
- `prefers-reduced-motion`: remove pulse animation entirely

## Copywriting Contract

Phase 1 has static/placeholder data. These are the copy strings to implement.

| Element | Copy |
|---------|------|
| Primary CTA (batch) | "Download ZIP" |
| Primary CTA (single file) | "Save file" |
| Optimize action | "Optimize" |
| Empty state — file list heading | "Drop images here" |
| Empty state — file list body | "SVG, PNG, JPEG, WebP, AVIF supported" |
| Empty state — formats line | "SVG · PNG · JPEG · WEBP · AVIF" (mono, muted) |
| Error status label | "Failed" |
| Processing status label | "Processing…" |
| Done status label | (no text — green dot only) |
| COOP error (console, not UI) | "[oimg] crossOriginIsolated is false. COOP/COEP headers are missing or a cross-origin resource is blocking COEP." |
| Theme toggle (dark→light) | aria-label: "Switch to light theme" |
| Theme toggle (light→dark) | aria-label: "Switch to dark theme" |
| Brand name | "OIMG" (JetBrains Mono, 600 weight, `letter-spacing: 0.04em`) |

**Destructive actions in Phase 1:** None. File removal UI is deferred to Phase 2+. No destructive confirmation copy needed for this phase.

**Tone:** Terse, developer-first. No marketing language. Labels are verbs or noun phrases. Error messages name the problem, not "something went wrong."
