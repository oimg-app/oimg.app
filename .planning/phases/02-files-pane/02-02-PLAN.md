The `id` originates from `STUB_FILES` (trusted authoring) but downstream consumers must treat it as opaque. |
| User-typed filter input | (Reserved for Phase 3 toolbar — `setFilter(q)`.) In Phase 2 the only writer is `setSortBy` from a fixed popover list of 5 enum values; no free-form text crosses any boundary. |

## STRIDE Threat Register (ASVS L1 scope)

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Tampering | `removeFile(id)` invoked from ContextMenu | accept | `id` is an opaque string from stub data; `removeFile` performs an in-memory filter with no side effects beyond the SPA tab. No undo per UI-SPEC §Destructive action — intended per design for the Phase 2 stub milestone. |
| T-02-07 | Information Disclosure | File names rendered in `FileRow` (`file.name`, `file.dim`) | mitigate | All values come from `STUB_FILES` (synthetic). React's default string interpolation auto-escapes HTML; no raw-HTML escape hatch is used anywhere in this plan. | T-02-08 | Spoofing / UI Redress | Programmatic `contextmenu` dispatch from `ctxbtn` | accept | The dispatch targets the row's own DOM node (`rowRef.current`) inside the same Radix `ContextMenu` subtree the user is already interacting with.
